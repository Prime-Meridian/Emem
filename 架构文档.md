# 喵喵机 - 技术架构文档

## 项目概述

**喵喵机**是一个基于 Web 的 AI 聊天应用，模拟微信界面，支持 PWA（渐进式 Web 应用），特别优化了 iOS 全屏体验。

### 技术栈
- **前端框架**: 原生 JavaScript (Vanilla JS)
- **数据库**: Dexie.js (IndexedDB 封装)
- **UI 风格**: 仿微信界面
- **PWA 支持**: Service Worker + Manifest
- **API 支持**: OpenAI 兼容 API + Google Gemini API

### 2. API 层 (API Layer)

#### 双 API 配置
- **主 API (Primary)**: 用于手动聊天
- **副 API (Secondary)**: 用于自动功能（朋友圈、公众号等）

#### API 类型支持
1. **OpenAI 兼容格式**
   - 标准 `/chat/completions` 端点
   - Bearer Token 认证

2. **Google Gemini API**
   - 自动检测 `generativelanguage.googleapis.com`
   - 消息格式自动转换
   - 支持 `systemInstruction` 参数

#### 核心函数
```javascript
// 获取 API 配置
function getApiConfig(isMainChat = true)

// 检测 API 类型
function detectApiType(apiUrl)

// 统一 API 调用
async function callApi(messages, isMainChat = true, options = {})

// Gemini 格式转换
function convertToGeminiFormat(messages)
function convertGeminiResponse(geminiData)
```

#### 屏幕切换
```javascript
function showScreen(screenId) {
    // 隐藏所有屏幕
    document.querySelectorAll('.screen').forEach(s => {
        s.classList.remove('active');
    });
    // 显示目标屏幕
    document.getElementById(screenId).classList.add('active');
}
```

### 4. 聊天系统 (Chat System)

#### 聊天类型
1. **单人聊天** (`type: 'single'`)
   - 一个 AI 人设 + 一个用户面具
   
2. **群聊** (`type: 'group'`)
   - 多个 AI 人设 + 一个用户面具
   - 支持 AI 之间互动

#### 聊天数据结构
```javascript
{
    id: 'chat_xxx',
    name: '聊天名称',
    type: 'single' | 'group',
    personas: {
        ai: { name, avatar, content },  // AI 人设
        my: { name, avatar, content }   // 用户面具
    },
    history: [
        { role: 'user', content: '...', timestamp: xxx },
        { role: 'assistant', content: '...', timestamp: xxx }
    ],
    isBlocked: false,  // 是否拉黑
    lastMessage: '...',
    lastMessageTime: xxx
}
```

#### 核心功能
- **消息发送**: `sendMessage()`
- **消息接收**: `receiveMessageHandler()`
- **消息编辑**: 支持编辑已发送消息
- **消息引用**: 支持引用回复
- **消息多选**: 批量删除/转发
- **拉黑功能**: `blockContact()` / `unblockContact()`
- **清空记录**: `clearChatHistory()`
- **搜索消息**: `openSearchChatModal()`

## 开发指南

### 添加新屏幕
1. 在 `index.html` 中添加 `<div id="xxx-screen" class="screen">`
2. 在 `styles.css` 中添加样式
3. 在 `scripts.js` 中添加逻辑和事件监听
4. 使用 `showScreen('xxx-screen')` 切换

### 添加新功能
1. 在 `KEYS` 中定义存储键
2. 使用 `dbStorage.get()` / `dbStorage.set()` 操作数据
3. 在 `appState` 中添加状态管理
4. 绑定 UI 事件

### 调试技巧
- 使用 `console.log()` 查看日志
- 检查 `db.kvStore` 查看存储数据
- 使用浏览器开发者工具的 Application 标签查看 IndexedDB

## 常见问题

### Q: 如何切换 API？
A: 进入设置 → API 设置，配置主 API 和副 API

### Q: 如何备份数据？
A: 使用浏览器开发者工具导出 IndexedDB，或使用应用内的导出功能

### Q: 如何添加新的 AI 人设？
A: 主屏幕 → 联系人 → 右上角 + → 添加 AI 人设

### Q: 为什么有些功能需要副 API？
A: 自动功能（朋友圈、公众号）使用副 API，避免消耗主 API 额度
